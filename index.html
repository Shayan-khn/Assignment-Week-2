Week 2 Task

Functional Requirements

  • System should provide signup and login options,as such, users have their own identity.
  • The system should store the user's signup details in a database.
  • The system should allow users to log in or link to other accounts such as Google or Facebook.
  • Password of the user should be stored in dashes to make it more secure.
  • It should allow users to change their passwords at any time. Users should also be able to change their password in case they forget it using their confirmed email address.
  • User's password details should be saved with proper encryption
  • If an incorrect password is entered, the system should notify the user with a message, and after the correct password, send a confirmation message to the user's other linked accounts before continuing.
  • After 60 minutes of inactivity, The system should be able to automatically log out a user.
  • An interface should be provided by the system where the users could write their queries.
  • User's queries should be encrypted as well to keep user privacy.
  • The system should use state-of-the-art algorithms to provide answers to user questions.
  • The system should display the results to the user on the screen.
  • The system should store a history of user queries and their responses.
  • The system should provide the user with a clear history button and permission to clear history, and require a confirmation message before clearing.
  • The system should keep improving the training data set for improved AI models.
  • The system should be able to update the saved panel code and constitution according to the changes.
  • Download button should be provided by the system, so that users can download the responses.
  • The system should provide a share button that allows users to share their results via a link.
  • Feedback form should be provided by the system for the users to provide feedback on the system.
  • The system should provide a multithreaded approach for multiple users to access the app concurrently.

Non-Functional Requirements

  • The system should protect user credentials.
  • The user's password should be hidden at login.
  • The system should protect the user's private information from unauthorized users.
  • System must be secure enough to prevent damage from unauthorized users.
  • The system should be immune to all kinds of attacks such as SQLI, DDOS, XSS.
  • Process of accessing user's sensitive data should be secured by the system.
  • Only system administrators can assign roles and change system permissions.
  • A flexible service-based architecture should be ensured by the system.
  • It should provide an easy-to-understand interface.
  • The system must be available all the time(24/7).
  • It should take less than a second for the system to update and save all the information.
  • Better component design should be provided to the users.
  • The system should be modular and open to change with new features.
  • The system must take less than a second after clicking a link in the navigation bar.
  • App should be supported on at least all popular operating systems.
  • In-app instructions should be easy for users to understand.
  • In case of maintenance, users must be notified in advance.
  • In case of maintenance, users must be notified in advance.
  • The App should contain minimum steps (around 3-4 clicks) to complete a task.
  • The system should perform at its optimum during peak hours or when multiple users are using it at the same time.
